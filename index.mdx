CalenderyBack is the server-side backbone of a social calendar application built with Spring Boot 4 and Java 21. It exposes a secure REST API plus a real-time STOMP/WebSocket channel for private messaging, backed by PostgreSQL and Supabase Storage for media. All protected endpoints use HTTP Basic auth with Spring Security role-based access control.

## Getting Started

<Steps>
  <Step title="Clone and configure">
    Clone the repository and set the required environment variables (`DATA_BASE_URL`, `SUPABASE_URL`, etc.) as described in the [configuration guide](/deployment/configuration).
  </Step>
  <Step title="Build and run">
    Build the Docker image and start the server on port 8080 with `docker compose up` or by running the JAR directly. See the [Docker deployment guide](/deployment/docker).
  </Step>
  <Step title="Register a user">
    Call `POST /api/users/auth/register` with an email and password. Confirm the verification email to activate the account.
  </Step>
  <Step title="Authenticate and explore">
    Pass `Authorization: Basic <base64(email:password)>` on every protected request. Explore the full [API Reference](/api/users/overview).
  </Step>
</Steps>

<Note>
  All endpoints under `/api/*/app/**` require a user with the `ROLE_USER` role. Public endpoints (registration, confirmation, login) are open without credentials.
</Note>
